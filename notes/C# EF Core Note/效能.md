# EF Core 效能

## 1. AsNoTracking：
- 預設情況下，EF Core 會追蹤查詢結果的實體狀態，會增加記憶體使用。如果只是讀取資料而不需要修改，可以使用 `AsNoTracking()` 減少記憶體使用。
```csharp
var users = _dbContext.Users.AsNoTracking().ToList();
```

## 2. 部分更新：
- 當更新實體時，EF Core 預設會更新所有欄位。若只需更新部分欄位，可以使用 `Attach` 實體附加，減少不必要的資料庫操作。
```csharp
var user = new User
{
    Id = model.Id,       // 主鍵一定要
    Name = model.Name
    Email = model.Email
};

_dbContext.Attach(user);

var entry = _dbContext.Entry(user);
entry.Property(x => x.Name).IsModified = true;
entry.Property(x => x.Email).IsModified = true;

_dbContext.SaveChanges();
```

## 3. 批次操作：
- 在需要批次新增、更新或刪除，使用單一的 `SaveChanges()` 呼叫來提交所有變更，避免多次呼叫。
```csharp
foreach (var user in newUsers)
{
    _dbContext.Users.Add(user);
    _dbContext.SaveChanges();
}
```
改為
```csharp
_dbContext.Users.AddRange(newUsers);
_dbContext.SaveChanges();
```

## 4. 使用原生 SQL 查詢：
- 對於複雜查詢或效能要求高的情況，可以使用 `FromSqlRaw` 或 `ExecuteSqlRaw` 來執行原生 SQL 查詢；或是透過取得目前 DbContext 使用的資料庫連線，作為 Dapper 或是其他支援原生 SQL 的工具查詢使用。

## 5. 注意欄位投影：
- 在查詢時只選取需要的欄位，避免載入不必要的資料。使用 `Select` 方法來投影所需欄位。
```csharp
var dict = _dbContext.Files
    .ToDictionary(x => x.Id, x => x.FileName);
```
改為
```csharp
var dict = _dbContext.Files
    .Select(x => new { x.Id, x.FileName })
    .ToDictionary(x => x.Id, x => x.FileName);
```

## 6. 確認實際執行的 SQL：
- 使用 `ToQueryString()` 方法來檢查 EF Core 產生的 SQL 語句。
- 在配置 DbContext 時使用 `LogTo` 也可以記錄執行的 SQL 語句，方便檢查。
```csharp
builder.Services.AddDbContext<XXXDbContext>(options =>
    options.LogTo(Console.WriteLine, LogLevel.Information)
);
```
