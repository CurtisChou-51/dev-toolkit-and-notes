# Visual Studio 建置屬性

- 紀錄一些因為檔案建置屬性不正確產生的問題

## 1. 程式碼檔案 (.cs) 未參與編譯

- 問題：新增的 .cs 檔案未被編譯進專案  
![](01.jpg)

- 檢查方式：檢視檔案屬性中的「建置動作」(Build Action)  
![](02.jpg)

- 解決方法：將建置動作設為「編譯」(Compile)  
![](03.jpg)

- 註：此區圖片為 Visual Studio 2017 所截取，可能與目前不同

## 2. 設定檔未複製到輸出目錄

- 問題：NLog.config 與報表範本檔案 .rdlc 檔未複製到輸出目錄

- 解決方法：將「複製到輸出目錄」設為「如果較新則複製」或「永遠複製」
![](04.png)
